# 팔자냥 PRD (Product Requirements Document)

> 버전: 1.0
> 작성일: 2025-01-15
> 상태: Draft

---

## 1. 제품 개요

### 1.1 제품명
**팔자냥** (임시)

### 1.2 한 줄 설명
₩1,500에 사주를 볼 수 있는 모바일 웹 서비스

### 1.3 제품 비전
- 저렴하고 접근성 높은 사주 서비스 제공
- 현대적이고 친근한 UX로 MZ세대 타겟
- 정확한 만세력 기반 + AI 해석으로 신뢰성 확보

---

## 2. 타겟 사용자

### 2.1 주요 타겟
| 구분 | 내용 |
|------|------|
| 연령대 | 20-30대 (MZ세대) |
| 성별 | 여성 중심 (포스텔러 기준 75%) |
| 지역 | 한국 → 미국 → 인도 (단계적 확장) |

### 2.2 사용자 페르소나
- 사주에 관심은 있지만 비싼 비용이 부담스러운 사람
- 점집 가기는 부담스럽지만 운세가 궁금한 사람
- 친구/연인과 궁합을 재미로 보고 싶은 사람
- 새해/생일 등 특별한 날 운세를 확인하고 싶은 사람

---

## 3. 핵심 기능

### 3.1 MVP (v1.0) 기능

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 개인 사주 | 사주팔자 + 대운 기반 인생 전체 분석 (성격, 기질, 재능, 진로) | P0 |
| 신년운세 | 세운 + 월운 기반 올해 월별 운세 분석 | P0 |
| 궁합 | 두 사람의 사주 궁합 분석 | P0 |
| 연애운 | 연애/결혼 관련 운세 | P0 |
| 만세력 미리보기 | 결제 전 사주팔자 무료 확인 (결제 유도) | P0 |
| 인스타 공유 | 결과 카드 이미지 공유 | P0 |

## 4. 사용자 플로우

### 4.1 메인 플로우
```
랜딩 페이지
    ↓ [사주 보러가기]
로그인/회원가입
    ↓
홈 화면 (4개 탭: 개인사주 | 신년운세 | 궁합 | 연애운)
    ↓ (개인사주 탭 기본 선택)
정보 입력 (생년월일시, 성별 등)
    ↓
결제 유도 페이지
├── 만세력 미리보기 (사주팔자, 오행 차트)
└── "전체 해석 보기" CTA
    ↓
결제 페이지 (코인 패키지 선택)
    ↓
결제 진행 (토스페이먼츠)
    ↓
로딩 (사주 분석 중...)
    ↓
결과 페이지
    ↓
[선택] 인스타 공유 → 다음 사주 무료 리워드
```

### 4.2 입력 정보

#### 개인 사주 / 단일 운세
| 필드 | 필수 여부 | 옵션 |
|------|-----------|------|
| 생년월일 | 필수 | - |
| 양력/음력 | 필수 | 양력, 음력, 윤달 |
| 성별 | 필수 | 남성, 여성 |
| 태어난 시간 | 선택 | 시간 선택 or "모름" |
| 관계 | 필수 | 본인, 여자친구, 남자친구, 아빠, 엄마, 형제, 자녀 등 |

#### 궁합
| 필드 | 필수 여부 |
|------|-----------|
| 본인 정보 | 필수 (위와 동일) |
| 상대방 정보 | 필수 (위와 동일) |

## 5. 수익 모델

### 5.1 코인 시스템

- 사주 1회 = 1코인 소모 (모든 유형 동일)
- 코인 패키지 구매 방식

### 5.2 가격 정책

| 패키지 | 가격 | 보너스 | 코인당 가격 |
|--------|------|--------|-------------|
| 1코인 | ₩1,500 | - | ₩1,500 |
| 5코인 | ₩6,500 | +0.5코인 | ₩1,182 |
| 10코인 | ₩12,000 | +2코인 | ₩1,000 |

### 5.3 리워드 정책

| 항목 | 리워드 | 비고 |
|------|--------|------|
| 인스타 공유 | 1코인 무료 | 계정당 월 1회 제한 |

### 5.4 결제 수단
- 토스페이먼츠 (카드결제)
- 카카오페이 (Phase 2)

## 6. 기술 스택

### 6.1 아키텍처
```
[Frontend]          [Backend]           [Database]
 Next.js    ←→    Python (FastAPI)  ←→   Supabase
    ↓                   ↓
 Vercel              Vercel/기타
                        ↓
                [사주 계산 엔진]
                 lunar-python +
            korean-lunar-calendar
                        ↓
                   [LLM API]
                  (해석 생성)
```

### 6.2 상세 스택

| 영역 | 기술 |
|------|------|
| Frontend | Next.js (React), TypeScript |
| Backend | Python (FastAPI) |
| Database | Supabase (PostgreSQL) |
| Auth | Supabase Auth |
| 결제 | 토스페이먼츠, 카카오페이 |
| 배포 | Vercel |
| 사주 계산 | lunar-python, korean-lunar-calendar |
| AI 해석 | LLM API (Claude/GPT) |

## 7. 사주 분석 방식

### 7.1 하이브리드 접근
```
[사용자 입력]
     ↓
[사주 계산 엔진] ← 정확한 알고리즘 (lunar-python)
     ↓
  사주팔자, 오행, 십신 등 계산 결과
     ↓
[LLM 해석] ← 자연스러운 문장 생성
     ↓
[결과 출력]
```

### 7.2 사주 계산 라이브러리
- **lunar-python**: 사주팔자, 오행, 십신, 대운 계산
- **korean-lunar-calendar**: 한글 간지 출력, 한국 천문연구원 기준 음력

### 7.3 개인사주 vs 신년운세 차이

| 구분 | 개인사주 | 신년운세 |
|------|----------|----------|
| 범위 | 인생 전체 | 올해 한 해 |
| 분석 기반 | 사주팔자 + 대운 (10년 주기) | 세운 (년운) + 월운 |
| 분석 내용 | 성격, 기질, 재능, 진로, 대운 흐름 | 월별 운세, 시기별 조언 |
| 예시 출력 | "리더십 있는 성격", "30대 후반 전환점" | "5월 금전운 좋음", "9월 이직 적기" |
| 용어 | 대운, 십신, 격국 | 세운, 월운, 행운의 달 |

## 8. 브랜딩 & 디자인

### 8.1 브랜드 아이덴티티

| 항목 | 내용 |
|------|------|
| 서비스명 | 팔자냥 (임시) |
| 캐릭터 | 고양이 캐릭터 (친근함) |
| 톤앤매너 | 친근함 기본, 안 좋은 내용은 진지하게 |
| 시각 스타일 | 미니멀 + 동양적 |

### 8.2 디자인 원칙
- 모바일 웹 최적화 (모바일 퍼스트)
- 인스타그램에 공유하고 싶은 결과 카드
- 깔끔하고 현대적인 UI
- 포스텔러 참고 (귀여운 일러스트 + 시각화)

### 8.3 결과물 형태
- 텍스트 기반 상세 해석
- 귀여운 일러스트/아이콘
- 오행 차트 등 시각적 요소
- 공유용 카드 이미지 자동 생성

## 9. 차별화 전략

### 9.1 경쟁사 대비 차별점

| 경쟁사 문제점 | 팔자냥 해결책 |
|---------------|---------------|
| 과도한 광고 | 광고 없음 (유료 모델) |
| 비싼 가격 | ₩1,500 파격 가격 |
| 추상적 풀이 | 구체적, 액셔너블한 조언 |
| 올드한 디자인 | 현대적, 미니멀 디자인 |
| 공유 어려움 | 인스타 공유 최적화 |

### 9.2 핵심 차별점
1. **₩1,500 가격** - 시장 최저가 (코인 패키지 시 더 저렴)
2. **현대적 디자인** - MZ세대 취향
3. **바이럴 기능** - 공유하면 무료 코인 리워드
4. **만세력 미리보기** - 결제 전 사주팔자 무료 확인
